마작 랭크(등급) 시스템
계획서

시즌(월간) 누적 승점으로 등급을 부여하고, 전용 랭크 페이지와 닉네임 옆 등급 뱃지로 노출한다.

누적 승점 등급 게임마다 저장(이벤트) 시즌/월간 리셋 전용 페이지 + 뱃지
Context

배경 & 목표

현재는 기간별 랭킹(/rank)과 개인기록만 있고 등급(티어) 개념이 없다. 승점(recordPoint)은 이미 게임마다 계산·저장되므로 이를 재료로 등급을 매긴다.

문제

실력/성취를 한눈에 보여줄 등급 체계 부재. 멤버 동기부여 약함.

해결

시즌 누적 승점 → 구간별 등급(S/A/B…). 매월 리셋되는 시즌 경쟁.

결과

전용 랭크 페이지 + 곳곳 닉네임 옆 등급 뱃지로 즉시 인지.

Requirements

확정 요구사항

① 누적 승점 등급

시즌 누적 recordPoint 합을 구간으로 잘라 등급 부여(단순).

② 게임마다 저장

기록 등록/수정/삭제 시 해당 회원 시즌 standing 갱신·저장 → 뱃지 조회 O(1).

③ 시즌/월간 리셋

시즌 = 월(YYYY-MM), 매월 자동 리셋 + 시즌별 기록.

④ 전용 페이지 + 뱃지

/rank-tier 페이지 + 닉네임 옆 등급 뱃지.

Architecture

전체 흐름

마작 기록 등록/수정/삭제 → RankTierRecalcEvent (AFTER_COMMIT, 비동기) → 시즌 승점 SUM 재계산 → STANDING upsert (점수·등급)
STANDING 테이블 → /rank-tier 리더보드 · 내 등급 진행바 → 닉네임 옆 등급 뱃지(/rank 등)

설계 원칙 — standing은 이벤트 시 SUM 재계산으로 upsert(증분가산 아님 → drift 없음). 소프트 삭제(delStatus='Y')도 재계산으로 자연 반영.

Logic

등급 산정 로직

단계내용
시즌 결정matchs.registDate → YYYY-MM (현재월 = 현재 시즌)
승점 집계해당 시즌 범위 + 삭제 안 된 대국의 recordPoint 합 (회원별)
등급 판정point ≥ MIN_POINT 만족하는 최상위 등급
저장STANDING upsert (point · gameCount · tierName 스냅샷)
진행바다음 등급 MIN_POINT − 현재 point = 남은 점수

토너먼트 기록 포함(기존 랭킹과 동일). 매월 자동 리셋(새 시즌 키 생성).

Rollout & Verify

실행 순서 & 검증

실행 순서

  1. create-rank-tier.sql 실행 (테이블·시드·백필·메뉴)
  2. 백엔드 재배포
  3. 프론트 npm install && npm run build 배포

검증

  • 기록 등록 → 4명 시즌 standing/등급 갱신
  • 삭제(soft) → 재계산으로 승점 감소
  • /rank-tier 리더보드·진행바, /rank 뱃지 노출
  • 월 경계 → 새 시즌 자동 생성

⚠️ 등급 임계값은 운영 데이터 보고 BGM_AGIT_RANK_TIER UPDATE로 튜닝(코드 변경 불필요).

Summary

요약

기존 승점 데이터 위에 시즌 누적 등급을 얹는다. 기록 이벤트로 standing을 재계산 저장하고, 전용 페이지와 닉네임 뱃지로 노출. 임계값·시즌은 운영에서 유연하게 조정.
